# SAAS 電子名片網站 PRD

## 1. 商業目標

這個 SAAS 要解決的不是「把紙本名片搬到網頁」，而是把每一次掃碼、點擊、加 LINE、詢問與預約變成可追蹤的商機。

核心成果：

- 增加名單：訪客留下姓名、電話、LINE 或需求。
- 增加成交機會：名片頁直接引導諮詢、預約、報價或加入官方帳號。
- 降低跟進成本：業務或店家能在後台看到誰來過、從哪裡來、點了什麼、目前跟進狀態。
- 沉澱資料：每張卡片、每個入口、每個活動 QR code 都能累積轉換數據。

## 2. 目標客戶

- 個人業務：保險、房仲、貸款、直銷、顧問、教練。
- 中小企業：每位員工一張卡，公司統一品牌與名單管理。
- 展會與活動團隊：用 QR code 收集攤位訪客與活動名單。
- 地方商家：讓客戶快速收藏、預約、加入 LINE OA 或索取優惠。
- 教育與講師：把課程簡介、報名、LINE 社群與講義入口集中在一頁。

## 3. 使用者角色

- 訪客：掃 QR code 或點分享連結，看名片並採取行動。
- 名片持有人：管理自己的照片、介紹、按鈕、社群與名單。
- 公司管理者：管理多位成員、品牌樣式、名單匯出與成效。
- 系統管理者：管理租戶、方案、用量、異常與支援。

## 4. 核心旅程

1. 公司建立租戶與品牌設定。
2. 管理者新增成員卡片。
3. 成員設定頭像、職稱、服務內容、CTA 按鈕與分享 slug。
4. 系統產生公開名片 URL 與 QR code。
5. 訪客掃碼進入名片頁。
6. 訪客點擊「加入 LINE」、「預約諮詢」、「索取報價」或「存入通訊錄」。
7. 系統記錄來源、點擊、表單資料與跟進狀態。
8. 管理者在 CRM 後台查看名單、篩選來源、匯出或標記跟進結果。

## 5. MVP 功能

### 必做

- 租戶與成員名片資料。
- 公開名片頁。
- CTA 按鈕管理。
- Lead 表單。
- 瀏覽與點擊事件記錄。
- 簡易 CRM 名單頁。
- 名單狀態：new, contacted, qualified, won, lost。
- 來源參數：source, campaign, medium, referrer。
- 手機優先 UI。

### 應做

- vCard 下載。
- QR code 匯出。
- 自訂網址 slug。
- 團隊名片列表。
- CSV 匯出。
- 基礎權限：owner, manager, member。

### 暫不做

- 複雜版型編輯器。
- 會員點數。
- 付款訂閱。
- AI 客服正式接管。
- 多國語系。
- 外部 CRM 雙向同步。

## 6. 成功指標

- 每張名片每週瀏覽數。
- CTA 點擊率。
- Lead 轉換率。
- 加 LINE 點擊數。
- 預約或詢價提交數。
- 管理者每週登入與匯出次數。
- 每位業務平均新增名單數。

## 7. 版本規劃

- V0：靜態原型，確認產品敘事與主要畫面。
- V1：Worker + D1 MVP，可建立卡片、公開瀏覽、收名單、看 CRM。
- V2：LINE LIFF/Login/Rich Menu 串接，記錄 LINE userId 與 postback source。
- V3：團隊、方案、品牌白標、QR code 批次產生。
- V4：AI follow-up、分眾廣播、外部 CRM 或行事曆串接。
